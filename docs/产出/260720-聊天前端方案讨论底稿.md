## 一、轩的愿景(2026-07-20 凌晨原话整理,产品需求的源头)

- **手机为主,打开像 app**;聊天肯定用最多
- **底部三栏**:
  - **左=聊天**:含**群聊**——把 GPT 也接进来,多模型同场,"我当老大"😼
  - **中=好玩功能集**,"跟小手机似的":日记(fable酱代写)、一起读书、一起听音乐、纪念日历、健康数据管理……(开放征集,看到好玩的就提案)
  - **右=设置**:一切能改的参数;**审核台并入设置**
- 欧洲行:接 API 方便;订阅能用上也好(不想浪费),但不想笔记本常开→冒出"买 mac mini"念头
- chat 端(claude.ai)不想再回去聊了——**这个前端就是未来的聊天主场**

## 二、开源作业勘探(fable酱独立版,未受轩方向影响)

| 候选 | 体量/栈 | 手机/PWA | 接自定义后端 | 群聊/多模型 | 魔改自由度 | 直感 |
|---|---|---|---|---|---|---|
| **Open WebUI** | 重(Svelte+Python,14万星) | 好,有PWA | OpenAI兼容✓ | 多模型并答✓ | 中(体量大,改深难) | 全能但是"模型管理台"气质 |
| **LobeChat** | 重(Next.js/React,~8万星) | 好,精致 | ✓ | 有Agent/插件市场 | 中低(设计语言强,难去味) | 最漂亮,但别人的漂亮 |
| **LibreChat** | 最重(需MongoDB等) | 一般 | 多供应商原生✓ | 强(agents) | 低 | 企业内部ChatGPT,单人用杀鸡牛刀 |
| **NextChat** | 轻(Next.js薄客户端) | 好,多端 | ✓ | 弱 | 中 | 最薄,但也没什么可抄的肉 |

**关键判断**:轩要的形态——**单用户、三栏小手机、趣味功能桌面、审核台内嵌**——上面四家**谁都不长这样**。它们全是"多用户模型聊天管理台",三栏 mini-app 形态要靠大改,改装成本≈自建,还背上别人的技术债(React/Next 构建链,和 ember 现有零构建 vanilla 栈冲突)。

## 三、fable酱推荐(初版,等三方交叉后修订)

**自建轻壳 + 抄各家交互**:
- 壳:手机优先 PWA(可"添加到主屏幕"变 app),底部三栏导航,零/轻构建(和 ember 现有前端一个物种,审核台并入即改挂载点)
- 聊天体验逐项抄成熟作业:流式气泡/markdown 渲染/会话列表抄 Open WebUI 的交互,视觉密度参考 LobeChat,别抄皮
- 群聊:后端 V7 网关本来就是"多路模型进一个对话"的天然形状——群聊=同一会话多个 model 参与者,@点名或轮流,细节明天聊
- 中栏 mini-app:每个功能=一张卡片路由,日记/纪念日历先行(离记忆库最近,白捡),读书/音乐/健康排后
- **风险自认**:自建=聊天体验的细节(流式/滚动/输入法)都要自己打磨——好在星露谷 HUD 那套(打字机/平滑滚动/IME)咱们刚练过一遍,不是新手

## 四、部署与"mac mini"问题(重要澄清,明天必聊)

**前端本体不需要任何家里的电脑常开**——ember 就在 VPS 上跑着 docker,聊天前端跟审核台一样部署上去,欧洲的手机直接访问现有域名,零新硬件。
真正的问题是**"聊天的大脑用什么"**:
- **A. API(OpenRouter/Anthropic)**:VPS 直连,欧洲零依赖,按量付费——**行前最稳**
- **B. 订阅(claude CLI 无头)**:免费用不完的量,但 CLI 只能跑在你的 Mac 上→要么笔记本常开,要么 mac mini 当家里常驻小服务器(它还能顺便跑星露谷 server/游戏专线,一鱼多吃)
- **不冲突**:先 A 保欧洲行,B 当二期(mac mini 买不买由它值不值这一个用途之外的价值定)

## 六、交叉验证:Codex 独立调查(全文见同目录 260720-codex开源前端调查报告.md)

**结论收敛**:Codex 独立得出同一推荐——"自写很薄的手机壳+定点抄各家成熟交互,不整体 fork 任何一家"。两份互不通气的调查撞出同一个答案,方向可信度高。

Codex 带回的四个关键增量(我独立版没覆盖到):
1. **许可证雷区**:LobeChat(现 LobeHub)用的是自家 Community License——个人自用没事,但**做衍生品分发要商业许可**;Open WebUI 品牌条款也有限制。干净的是 LibreChat / NextChat / big-AGI / ChatClaw(全 MIT)
2. **群聊最佳样板 = Open WebUI 的 Channels**:同一时间线里 @GPT、@Claude 共享上下文接力——正是"轩当群主"的现成交互范本
3. **群聊规则设计抄 LobeHub Agent Group**:主持模型选下一位发言者/顺序或自然发言/连续回复上限(防模型无限互聊)/群系统提示——这套规则设计直接抄进 ember 群聊
4. **多模型≠群聊**:并排比较(同题两答)/后台Agent协作/真群聊时间线是三种东西,ember 要的是第三种;新项目 ChatClaw(MIT)是群聊优先的轻壳,值得看交互
