## 架构

```
浏览器（GitHub Pages 上的 index.html）
   │  POST { messages, model, password }
   ▼
Cloudflare Worker（ember-proxy）
   │  1. 校验访问密码（对照 Secret ACCESS_PASSWORD），错就 401
   │  2. 加上 Authorization: Bearer <OPENROUTER_API_KEY>（Secret）
   │  3. 转发，开 stream:true
   ▼
OpenRouter（/api/v1/chat/completions）
   ▼
Claude Opus 4.6（anthropic/claude-opus-4.6）
   │  SSE 流式返回
   ▼ 原样透传回浏览器，前端逐字渲染（打字机效果）
```

- **为什么要 Worker 当中间人**：API key 不能放前端（会被偷）。Worker 在服务端偷偷加 key，前端永远看不到。
- **密码门禁的必要性**：CORS 只防浏览器跨站，挡不住 curl 直连 Worker。所以加一道密码校验，没对密码一律 401，绝不调用 OpenRouter。

## 技术栈和配置

- **前端**：原生 HTML/CSS/JS 单文件（index.html），无框架、无依赖。
- **代理**：Cloudflare Worker（Module 格式，worker/worker.js）。
- **模型**：OpenRouter 的 `anthropic/claude-opus-4.6`（注意是点不是横杠）。
- **wrangler.toml**：name=`ember-proxy`，main=`worker.js`，compatibility_date=`2024-11-01`。
- **前端顶部可配置常量**（index.html）：
  - `WORKER_URL = "https://ember-proxy.claude-bflwhp.workers.dev"`
  - `MODEL = "anthropic/claude-opus-4.6"`
- **Worker 顶部常量**（worker.js）：
  - `ALLOWED_ORIGIN = "https://cloudxuan1.github.io"`（CORS 放行来源，必须等于 Pages 域名）
  - `DEFAULT_MODEL = "anthropic/claude-opus-4.6"`

## 部署和运维

- **前端（GitHub Pages）**：main 分支根目录，开在 Settings→Pages。改完 push 到 main 自动重新 build（1-3 分钟）。
- **Worker 部署**：Cloudflare 后台 → ember-proxy → Edit code 粘贴 worker.js → Deploy；或 `cd worker && npx wrangler deploy`。
- **两个 Secret**（Cloudflare → Worker → Settings → Variables and Secrets，类型必须选 Secret）：
  - `OPENROUTER_API_KEY`：OpenRouter 的 key
  - `ACCESS_PASSWORD`：访问密码（前端打开时要输的那个）
  - ⚠️ 没设 ACCESS_PASSWORD 时 Worker 对所有请求返回 401（fail-safe，宁可全拒不裸奔）。
- **Cloudflare 账号**：Claude.bflwhp@gmail.com
- **Worker 网址**：https://ember-proxy.claude-bflwhp.workers.dev

- **字号偏离设计稿**：设计稿正文 12px，聊天界面放大到正文 15px、输入框 16px。原因：聊天要读长文 + iOS 输入框 <16px 聚焦会自动放大页面。
- **布局**：手机优先，单列。顶栏（spark+ember）/ 消息流 / 底部输入栏。用户气泡靠右暖棕，AI 气泡靠左白卡。
- **密码界面**：复用怀旧版卡片样式（白卡+轻阴影+serif 的 ember），全屏遮罩居中。

## 数据库

无。对话历史只存浏览器内存（index.html 的 `messages` 数组），刷新/关页面即清空。localStorage 只存访问密码（键 `ember_pw`），不存对话。

## 绝对不能动（完整清单）

1. **API key 和访问密码绝不写进代码**，只放 Cloudflare Secret（`OPENROUTER_API_KEY`、`ACCESS_PASSWORD`）。
2. **Worker 的 ALLOWED_ORIGIN 必须等于前端 Pages 域名**，否则浏览器 CORS 拦截；换域名要同步改 + 重新 Deploy。
3. **改 worker 后必须重新 Deploy**，本地改不影响线上。
4. **模型 slug 是 `anthropic/claude-opus-4.6`（点，不是横杠）**，OpenRouter 上没有 claude-opus-4-6。
5. **仓库已 public**——commit 任何东西前确认不含 key/密码/隐私。

## 经验教训

- **OpenRouter 模型 slug 用点不用横杠**：是 `anthropic/claude-opus-4.6`，任务里写的 `claude-opus-4-6` 不存在会报 400。上线前 `curl openrouter.ai/api/v1/models` 核实过。
- **CORS 不是安全防护**：CORS 只在浏览器里生效，curl/脚本直连 Worker 能绕过。只锁 CORS 域名挡不住盗用，必须加密码校验。
- **GitHub 免费版不给 private 仓库开 Pages**（报 422）。本项目把仓库改 public——因为 key/密码都不在代码里，公开无敏感泄露。
- **Python 3.14 在 Claude.app 沙箱里 `http.server` bind 失败**（进程在但端口不监听）：本地预览 server 起不来，改用离线 node 测试 + 真实 worker curl 验证。
- **密码方案是简化版门禁**：单一共享口令 + Worker 端明文 Secret 比对，非正规账号体系（正规应存哈希 + 各自账号 + 登录态）。对个人项目防白嫖够用。
